Issue #15 — The supplier becomes the competitor

Apr 14 → 21
The big story

Anthropic launches Claude Design — and Figma stock drops 7%

Three events happened in three days. Anthropic's CPO Mike Krieger quietly stepped off Figma's board on April 14. Claude Opus 4.7 shipped on April 16 with high-resolution vision and an 87.6% SWE-bench score. Claude Design launched on April 17 — a tool that turns prompts into prototypes, slides, one-pagers, and implementation bundles that feed directly into Claude Code. Figma lost 7% of its market value within hours. The competitive logic is stark: Figma pays API fees to Anthropic for Sonnet 4.5; Claude Design runs Opus 4.7 internally at no inference cost. "It's very difficult to compete on AI against the company that provides your AI inference," one analyst wrote.

TechCrunch VentureBeat Gizmodo 📉 Figma -7% 💰 Anthropic $30B ARR 📅 Apr 17

Mike Krieger exits Figma's board — the tell

Instagram co-founder, Anthropic CPO. Left the board on April 14 — three days before Claude Design launched. Analysts called it "a canary in the coal mine for SaaS." Frontier labs are no longer content to be model suppliers. They want the application layer. The pattern is now visible across developer tools, search, office software, and design.

TechCrunch 📅 Apr 14 📉 -9% on announcement

Figma's quiet pivot: Gemini, OpenAI — not Anthropic

Dylan Field posted on LinkedIn about Claude Sonnet 4.5 in Figma Make — but the broader picture is sharper. Figma's AI now runs on Gemini 3.0 Pro and OpenAI GPT Image models. They're not buying Opus 4.7. Cursor is adding Kimi K2.5 from China. The companies that built on Anthropic are quietly decoupling from the company that just became their competitor.

VentureBeat 🔄 strategic decoupling

Our read: This is the SaaS dependency trap made visible in one week. Figma, Lovable, and Cursor all built on Anthropic's models — and now Anthropic has walked through the door they built. The "SAASpocalypse" thesis says frontier labs eventually consume the apps they serve. This week gave us the first clean example.

Reddit's verdict on Claude Design: "Resounding meh" — with caveats

The r/ClaudeAI launch reaction landed somewhere between skepticism and frustration. Token burn was the #1 complaint: 2–3 full prompts can exhaust weekly Pro limits, and one user reported hitting their session cap in under 10 minutes on launch day. The aesthetic critique was consistent — uniform serif fonts, identical blinking status dots, "container soup" card layouts. The real signal buried in the thread: non-developers are discovering Claude for the first time through Design demos. "I shipped a SaaS in one Claude Design chat" posts started appearing within 24 hours. That's a new audience, not the existing user base.

Twitter split: "RIP designers" vs "another AI slop feature"

Demos framed as "killing Figma" drove the biggest organic reach all week. The people most excited weren't designers — they were founders and PMs who never used Figma, seeing prototype-quality output for the first time. Designers themselves were divided: some saw the implementation bundle as the real story; others dismissed it as a high-end template generator. The gap in reactions tracks almost perfectly with seniority.

Tools & workflows

The loop is closed: Claude Code → Figma → Claude Design → Claude Code

The Code to Canvas integration finally has a full circuit this week. Type "Send this to Figma" in Claude Code — your running UI converts to editable Figma layers in about 30 seconds. Iterate in Figma Make. Pass the result to Claude Design as a reference. Export the implementation bundle. Back to Claude Code. This bidirectional loop — code → canvas → code — closes the designer-developer handoff argument. For small product teams who can't afford a dedicated handoff process, this is now the default workflow.

Figma Blog Muzli Builder.io 🔄 bidirectional loop

Claude Design vs Figma Make: which one actually fits

Both take natural language and produce a usable design — but from opposite directions. Claude Design extends the agentic workflow (conversational, Opus 4.7, implementation bundle output). Figma Make layers AI onto an existing canvas (accessible, team-shareable, integrates with components you already have). The practical split: use Claude Design starting from scratch, shipping to code. Use Figma Make iterating inside a live design system.

The 3 Claude Code + Figma workflows practitioners actually use

UX Planet broke down the three patterns that have stuck: 1) Design-to-code (Figma → Claude Code via MCP), 2) Code-to-canvas (Claude Code → Figma via "Send this to Figma"), 3) Full loop (both directions in the same session). The third is what most teams are building toward — and what Code to Canvas + Claude Design makes possible for the first time.
